A step-by-step installation guide for the COGENSZ Portfolio — a high-performance, cinematic portfolio built with Vite, React 19, Tailwind CSS v4, and Framer Motion. Follow this guide to get the project running on your local machine in under 5 minutes.

Installation

03
1

Clone the Repository

Download the project source code from the COGENSZ portfolio repository.

bash 
git clone https://github.com/cogensz/cogensz-portfolio.git
cd cogensz-portfolio
2

Install Dependencies

Install all packages defined in package.json — React, Framer Motion, Tailwind, and all dev tools.

bash 
npm install
3

Start the Development Server

Launch Vite's hot-module-replacing dev server. Changes reflect instantly in the browser.

bash 
npm run dev

# Expected output:
  VITE v8.0.0  ready in ~300 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
✦  Open http://localhost:5173 in your browser. The page hot-reloads automatically on every file save — no manual refresh needed.
4

Build for Production (optional)

When ready to deploy, generate an optimised production bundle in the /dist folder.

bash 
npm run build     # compile & optimise → /dist
npm run preview   # preview the production build locally

Environment Variables

05

No variables are required by default. If you extend the project with a contact form or CMS, create a .env file at the project root:

.env 
# Never commit this file — it is listed in .gitignore
VITE_API_URL=https://your-api-endpoint.com
VITE_EMAILJS_SERVICE_ID=your_service_id
VITE_EMAILJS_TEMPLATE_ID=your_template_id
VITE_EMAILJS_PUBLIC_KEY=your_public_key
✦  All Vite environment variables must be prefixed with VITE_ to be exposed to the browser. Never put secret keys in client-side code.
